# 信源两层筛选（retrieval 阶段参考）

> 迁移自 v1.0「信源筛选」，并按 2.0 改造：从"对一批候选统一打分"改为**边检索边粗筛**——每个渠道返回候选后即时过两层，决定哪些值得深读，避免对低质源浪费昂贵抓取。

## 信度按来源类型（与渠道无关）
判断一条候选的来源信度，看它**是什么类型的内容**，而非来自哪个渠道：
- 一手：官方披露/财报/招股书/官网参数页/当事人原帖
- 权威二手：行业报告、专业媒体深度、研报、权威数据库
- 社群一手体验：真实用户测评/实拍/亲历吐槽（高具体性者）
- 聚合/转述：二次汇编、营销稿、内容农场（最低，常可弃）

**跨 2 类来源信度交叉验证**：同一子问题的深读源，尽量不全是同一类型（如官方+社群、权威二手+社群体验）。

## 第一层 · 方向粗筛（direction screen，快、便宜）
只看 Title / Snippet / URL / Date，不抓正文，逐条判：
- **方向一致性**：是否命中该子问题的 A/B/C 话题方向与核心概念？跑题的直接弃（`direction_pass=false`）。
- **明显噪音**：营销软文、纯导购、农场拼凑、与主体同名异指——弃并记 `drop_reason`。
- **时效软门槛**：默认倾向当前日期前约 1.5 年内；超期的不硬弃，但需更强的"权威/独到"理由才进第二层。

## 第二层 · 质量与信度分级（quality grade，对通过第一层者）
对通过方向粗筛的候选，按三个质量信号 + 来源信度综合给档（高/中/低）：
- **信息密度**：单位篇幅含多少可用事实/数据，而非空话。
- **具体性**：有无具体数字、案例、机制描述（vs 泛泛而谈）。
- **独到性**：是否提供别处没有的视角/一手信息（vs 人云亦云的转述）。
- **来源信度**：见上面的类型分级。
- **转载去重**：内容高度重复的多条，只留信度最高的那条，其余记 `duplicate_of`。

> 修 v1.0 旧问题：不再用"关键词绝对出现次数"打分（长水文会赢），改看密度×具体性×独到性；最终深读清单**不打散原文顺序**——顺序留给下游审计按原文位置处理。

## 选取规则（控成本）
- 每个**子问题维度**最多深读 `budget.max_sources_per_dimension`（默认 2）个：在该 qid 跨渠道的全部通过候选里，按质量档 + 信度 + 跨类型互补，挑 2 个 `selected_for_read=true`。
- 宁可少读高质的，不要多读低质的。某 qid 若通过候选不足 2 个，如实记录"信源不足"，留给审计判断是否回环——**不要为凑数深读低质源**。
- 深读=对选中源发 read 操作（jina 读网页 / 渠道的 note/post 读原帖），取回正文供下游审计清洗提炼。
